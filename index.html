About AES

What is AES?

Advanced Encryption Standard (AES), also known as Rijndael, is a symmetric block encryption algorithm (128 â€” bit block size, 128/192/256-bit key) adopted as an encryption standard by the US government based on the results of AES competition. This algorithm is well analyzed and is now widely used, as it was with its predecessor DES.

Definitions

Block the sequence of bits that make up input, output, State, and Round Key. Block can also be understood as a sequence of bytes
Cipher Key a secret cryptographic key that is used by the Key Expansion procedure to produce a set of keys for rounds (Round Keys); it can be represented as a rectangular array of bytes with four rows and Nk columns
Ciphertext output data of the encryption algorithm
Key Expansion procedure for generating Round Keys from Cipher Key
Round Key Round Keys are obtained from Cipher Key using the Key Expansion procedure. They are applied to State for encryption and decryption
State intermediate encryption result, which can be represented as a rectangular array of bytes with 4 rows and Nb columns
S-box a non-linear substitution table used in multiple byte substitution transformations and in the key extension procedure for one-to-one byte value substitution. The pre-calculated s-Box can be seen below
Nb the number of columns (32-bit words), State the condition. For AES, NB = 4
Nk the number of 32-bit words that make up the cipher key. For AES, NK = 4, 6, or 8
Nr number of rounds, NK NK and NB. For AES, NR = 10, 12, 14
Rcon[] an array with 32 32-bit words and is a constant for a given round.

Auxiliary procedures

AddRoundKey() transformation during encryption and reverse encryption, in which the Round Key XOR'itsya c State. The length of the RoundKey is equal to the size of State (that is, if Nb = 4, then the length of the RoundKey is 128 bits or 16 bytes)
InvMixColumns() transformation during decryption, which is the reverse of MixColumns()
MixColumns() an encryption transformation that takes all State columns and mixes their data (independently of each other) to get new columns
ShiftRows() encryption transformations that process State by Cycling the last three rows of State by different values
SubBytes() encryption transformations that process State using a non-linear byte substitution table (S-box), applying it independently to each state byte
